# HeyCoder

> Hey，coder——每天多一点灵感。不贩卖焦虑，只分享一点点：开源项目、开发者工具与技术趣闻，轻松读、慢慢攒。

**仓库**：[github.com/tageecc/heycoder](https://github.com/tageecc/heycoder)

---

## 这是什么

**HeyCoder** 是一个面向开发者的**开源与技术内容合集**：把值得一试的仓库、好用的小工具和可读性强的技术话题整理成 Markdown，托管在本仓库，方便检索、引用与协作。

你可以把它当成「个人向的 HelloGitHub 式清单」——**不追求大而全，而追求好读、可上手、多一点灵感**。

---

## 背景

开源项目每天都在更新，但多数人没有整块时间「逛 GitHub」。公众号适合**每天推一点**，仓库适合**沉淀、版本管理与社区贡献**——两者加在一起，既能坚持输出，又能留下可追溯的原文与链接。

本仓库就是 HeyCoder 的**公开资料库**：内容与公众号同调，也欢迎你在 Issue 里推荐或自荐项目。

---

## 愿景

- **每天分享一点，每天成长一点**——节奏轻，不卷字数。
- **多一点灵感，少一点焦虑**——偏有趣、偏实用，少制造 FOMO。
- **连接同路人**——好的项目值得被看见，好的分享值得被记录。

## 微信公众号

**HeyCoder** 已开通。若你更习惯在微信里阅读、接收推送：

- 打开微信 → 搜索 **「HeyCoder」** → 关注即可。

若你希望在本 README **顶部展示关注二维码**（方便从 GitHub 一键扫码），可将二维码图片放到仓库 **`assets/wechat-qrcode.png`**（建议正方形、清晰、背景简洁），提交后可在下方自行增加一行图片引用，或把图片发我代为接入：

```markdown
![HeyCoder 微信公众号](assets/wechat-qrcode.png)
```

（当前仓库未默认包含该图片，避免占位图误导。）

## 内容在哪里

所有正文在 [`content/`](content/) 目录下，按文件更新即可；不必在 README 里维护「第几期」表格。新文章可沿用 `HeyCoder001.md` 这类命名，或按日期命名，随你习惯。

## 参与
